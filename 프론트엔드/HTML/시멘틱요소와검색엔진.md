# Semantic Web
대부분의 사용자들은 인터넷에서 원하는 정보를 찾기 위해서 Naver, Google같은 검색 엔진을 사용한다. 따라서 웹사이트의 사용자 수를 늘리기 위해서는 검색엔진에 노출 되는 것이 굉장히 중요하다. 아무리 좋은 서비스를 개발했다고 하더라도 사용자에게 노출이 되지 않는다면 쓸모가 없기 때문이다.

검색엔진에 노출이 되기 위하여 SEO(검색엔진 최적화: Search Engine Optimization)같은 마케팅 도구를 사용하여 본인의 웹사이트가 검색되기 알맞은 구조로 웹사이트를 조정하기도 하는데, 이것은 기본적으로 검색엔진이 웹사이트 정보를 어떻게 수집하는지 아는 것으로부터 시작된다.

검색엔진은 로봇이라는 프로그램을 이용하여 매일 전세계 웹사이트 정보를 수집한다.(이것을 **크롤링**이라고 하며 검색엔진의 크롤러가 이를 수행한다.)그리고 검색 사이트 이용자가 검색할 만한 키워드를 예상하여 키워드에 대응하는 인덱스를 만든다.(이것을 **인덱싱**이라고 하며 검색엔진의 인덱서가 이를 수행한다.)

인덱스를 생성할 때 사용되는 정보는 검색엔진이 수집한 정보인데 이는 결국 **HTML코드**이다.즉, 검색엔진은 HTML코드 만으로 그 의미를 이해해야 하는데 이때 **시멘틱요소**를 해석하게 된다.

HTML로 작성된 문서는 컴퓨터가 해석하는 메타데이터와 사람이 사용하는 자연어 문장이 뒤섞여 있다.

```html
<font size="6"><b>Hello</b></font>
<h1>Hello</h1>
```
위 코드를 브라우저에서 실행하면 동일한 외형을 가지는데 이는 h1요소의 스타일이 1행과 동일하기 때문이다.

하지만 1행의 요소는 어떤 의미도 가지고 있지 않다. 다만 브라우저에게 폰트의 사이즈와 볼드체를 지정하는 메타데이터 만을 알리고 있다.그러나 2행의 요소는 제목중 가장 상위레벨이라는 의미를 내포하고 있기 때문에 **개발자가 의도한 요소의 의미가 명확하다. 이것은 코드의 가독성을 높이고 유지보수를 쉽게 만든다.**

시맨틱 요소로 구성되어 있는 웹페이지는 검색엔진에 보다 의미론적으로 정보를 전달 해 줄수 있기 때문에 검색엔진 또한 효과적인 크롤링과 인덱싱이 가능해졌다. **시멘틱 태그는 브라우저, 검색엔진, 개발자 모두에게 콘텐츠의 의미를 명확하게 설명하는 역할을 한다.**

**시맨틱 웹이란 웹에 존재하는 수많은 웹페이지들에 메타데이터를 부여하여, 기존의 잡다한 데이터 집합이었던 웹페이지를 '의미'와 '관련성'을 가지는 거대한 데이터베이스로 구축하고자 하는 발상이다.**

### HTML5 시맨틱 요소

tag | description
--|--
header | header
nav | 내비게이션
aside | 사이드에 위치하는 공간
section | 본문의 여러 내용을 포함하는 공간의미.
article | 본문의 주 내용
footer | footer
